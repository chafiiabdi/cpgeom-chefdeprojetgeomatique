# Contexte

Bienvenue dans la documentation **CPGEOM - Chef de projet géomatique**.

Ce site rassemble les notes, méthodes, ressources et travaux pratiques utiles pour suivre un projet géomatique : cadrage, données, qualité, formats, télédétection, automatisation et interopérabilité.

```{important}
Pour modifier le contenu, travaille dans les fichiers Markdown du dossier `source/`. Le dossier `docs/` contient uniquement le site HTML généré.
```

## Objectifs du site

| Objectif | Rôle |
|---|---|
| Centraliser les cours | Retrouver rapidement les notions importantes. |
| Structurer la formation | Séparer les grands blocs et leurs sous-blocs. |
| Faciliter les modifications | Modifier des fichiers `.md` simples à lire. |
| Préparer la publication | Générer le HTML dans `docs/` pour GitHub Pages. |

## Organisation générale

| Bloc du menu | Contenu principal |
|---|---|
| [Gestion de projet](gestion-projet.md) | Méthodes de pilotage, périmètre, tâches, risques et gouvernance. |
| [Base de données](base-donnees.md) | Introduction BDD, administration PostgreSQL et PostGIS. |

| [Données & Interopérabilité](donnees-interoperabilite.md) | VRT, droits, cycle de vie et métadonnées. |

## Comment lire la documentation

- Le menu latéral donne l'accès aux blocs principaux.
- Chaque bloc possède sa propre page d'introduction.
- Les sous-blocs ont chacun une page séparée pour ajouter les cours progressivement.
- Les tableaux servent de repères rapides.
- Les blocs `note`, `tip` et `important` mettent en avant les informations utiles.
